## spec·规格

- spec=specification，规格、规定、说明书、详细规则  
  
词语使用例句：

```
building spec = 建筑施工规范  
product spec = 产品规格  
meet the spec = 符合规格  
out of spec = 不符合规格  
write the spec = 写规范/规格说明  
```

---
### 计算机/工程领域

#### 1. 技术规则（Software/Protocol Spec）

示例：

```text
- HTTP/2 Spec
- LoRaWAN Regional Parameters Spec
- USB-C Electrical Spec
- JSON Spec
- Bluetooth  Core Spec
```
  
这些文件都定义了“一个系统怎样工作”：

- 明确格式
- 明确边界条件
- 明确设备/软件必须遵守的行为

———

#### 2. 产品参数（Engineering/Product Spec）

比方说某个设备的：

```text
- CPU 速度
- 内存大小
- 射频芯片型号
- 频率范围
- 功耗
- 尺寸和重量
``` 

也就是说明书的设备参数页（specsheet/datasheet）。

———

#### 3. 行为约束（系统行为的硬规则）

比方说：

```
- SF/BW/CR 的有效范围是 spec
- Duty cycle 限制来自法规 spec
- 路由 TTL、重传策略是协议 spec
- MCU 的 RAM/Flash 分布是硬件 spec
```
  
这些描述属于系统运行的底层规则。

#### GPT给了些工程师例句😁

这个词在工程师嘴里的语气，往往比条文更犀利。

> “Doesn’t meet spec.”

翻译：这个东西不达标，别上线。

>“Write a spec first.”

翻译：别乱写代码，先把需求文档写清楚。

> “We need to update the spec.”

翻译：架构变了，之前的文档全废了。

> “Which spec are you following?”

翻译：你跟我用的不是同一个世界线。
